## ЁЯЪА Quick Start

```bash
git clone https://github.com/Harsh-Codes-77/context-bridge.git
cd context-bridge

# Windows:
install.bat

# macOS:
chmod +x install-mac.sh && ./install-mac.sh

# Linux:
chmod +x install-linux.sh && ./install-linux.sh

# Then:
cb init
cb status
cb web

# Optional diagnostics:
cb doctor
```

`cb init` saves tokens to `~/.context-bridge/.env`. **Note:** Only the GitHub token is required! Linear and Slack tokens are completely optional and can be freely skipped.
You can manage repos anytime with `cb repo` тАФ no need to re-enter tokens.

### `cb status`
Fetches and displays your complete current context - branch, PR, ticket, CI, Slack - all at once.

```bash
cb status
```

**What it shows:**
- Current git branch
- Linked Linear/Jira ticket (auto-detected from branch name)
- GitHub PR status + unresolved comments
- Latest CI/CD run result (pass/fail + error location)
- Recent relevant Slack messages

**Under the hood:**
- тЪб **Parallel Execution**: Uses `ThreadPoolExecutor` to fetch GitHub, Linear, and Slack data simultaneously, drastically reducing fetch time.
- ЁЯЫбя╕П **Fault Tolerance**: If one integration fails (e.g., Slack token expired), the other integrations will still load successfully, and you'll see a clear warning instead of a crash.

**Options:**
- `--json`: Outputs a clean JSON object instead of rich terminal formatting. Perfect for scripting and piping into other tools (e.g., `cb status --json | jq '.github.ci_status'`). Missing tokens or API errors will gracefully output an `"error"` field within the JSON instead of breaking the execution.

### Setting Up Tokens

*(Note: Only the GitHub token is required! Linear and Slack tokens are completely optional, and their summaries will dynamically hide from the dashboard and CLI if not provided.)*

<details>
<summary>ЁЯУН GitHub Token</summary>

1. Go to [github.com/settings/tokens](https://github.com/settings/tokens)
2. Click **"Generate new token (classic)"**
3. Select scopes: `repo`, `workflow`
4. Copy the token тЖТ paste in `cb init`

</details>

<details>
<summary>ЁЯУН Linear Token</summary>

1. Open Linear тЖТ Settings тЖТ API
2. Click **"Create new API key"**
3. Copy the key тЖТ paste in `cb init`

</details>

<details>
<summary>ЁЯУН Slack Token</summary>

1. Go to [api.slack.com/apps](https://api.slack.com/apps)
2. Create a new app тЖТ **"From scratch"**
3. Add OAuth scopes: `channels:read`, `channels:history`, `groups:read`, `groups:history`, `users:read`
4. Optional (for better search results): `search:read`
5. Install app to workspace
6. Copy **Bot User OAuth Token** (or a user token with search permissions) тЖТ paste in `cb init`

If `search.messages` is unavailable for your token type, context-bridge automatically falls back to channel history scanning.

</details>

## ЁЯЦея╕П Web Dashboard

Run `cb web` to get a beautiful local dashboard at `http://localhost:4242`.

**Dashboard features:**
- ЁЯМ┐ All your active branches as cards
- ЁЯУК GitHub PR, CI status, Linear ticket, and Slack context per branch
- тП▒я╕П Last active timestamps plus cache age visibility

- ЁЯФД Auto-refresh (sessions every 30s, full context every 120s)

### Dashboard Search & Filter

**Find branches instantly** when you have 10+ active sessions:

- ЁЯФН **Live Search**: Type to filter branches in real-time (no page reload)
- тМия╕П **Keyboard Shortcut**: Press `/` anywhere to focus the search bar
- ЁЯОп **Smart Matching**: Filters by branch name, repo name, or files touched (case-insensitive)
- тЬи **Highlighted Results**: Matched text is highlighted in cyan for easy scanning
- ЁЯУК **Result Counter**: Shows "Showing X of Y branches" while filtering
- тЬХ **Clear Button**: Click `├Ч` to reset search instantly
- ЁЯТм **Server Fallback**: GET `/api/search?q=<query>` endpoint available for larger datasets

**Example:**
```
тЖТ Press "/" to focus search
тЖТ Type "auth" тЖТ instantly shows only branches matching "auth"
тЖТ Type "services" тЖТ filters to branches with service-related files touched
тЖТ See "Showing 3 of 12 branches" while filtered
тЖТ Click ├Ч or clear input тЖТ back to full view
```

### Dashboard Keyboard Shortcuts

**Power users**: Navigate and control the dashboard entirely with your keyboard. No mouse needed.

#### Global Shortcuts

| Key | Action |
|-----|--------|
| **r** | Refresh all context data (shows brief indicator) |
| **/** | Focus search bar instantly |
| **Esc** | Clear search input OR close any dialog/modal |
| **?** | Open keyboard shortcuts help overlay |

#### Card Navigation

| Key | Action |
|-----|--------|
| **j** or **тЖУ** | Move focus to next branch card |
| **k** or **тЖС** | Move focus to previous branch card |
| **Enter** | Expand focused card to show full details |
| **c** | Copy branch name of focused card |
| **g** | Copy git checkout command of focused card |
| **Delete** or **x** | Prompt to delete focused card's session |

#### Visual Feedback

- **Focused card** gets a bright cyan highlighted border (distinct from mouse hover)
- **Position indicator** shows "3 / 12" in bottom right when navigating with keyboard
- **Smooth scroll** keeps focused card visible and centered
- **Footer hint** displays "Press ? for keyboard shortcuts" for discoverability

#### Tips

- тЬЕ Shortcuts **don't fire** while typing in the search box
- тЬЕ All shortcuts work in **one clean event listener** (no interference)
- тЬЕ Works great with **tmux/screen** or remote SSH sessions
- тЬЕ Press **?** anytime to see the help overlay with all available shortcuts

**Example workflow:**
```
тЖТ Press "/" тЖТ Type "auth" тЖТ See matching branches
тЖТ Press "j" to move down тЖТ Focus on "fix/auth-timeout"
тЖТ Press "g" тЖТ Copies "git checkout fix/auth-timeout"
тЖТ Paste in terminal, continue coding
тЖТ Back in dashboard, press "r" тЖТ Refreshes all context
тЖТ Done without touching the mouse once!
```
